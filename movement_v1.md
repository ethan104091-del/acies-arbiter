## 0. 設計原則

移動由 7 個確定量決定：
1. 兵種基礎速度
2. 地形類型
3. 道路網狀況
4. 道路擁塞
5. 天氣（連結 weather_v1.md）
6. 疲勞累積
7. 車輛/裝備狀態

**不擲骰、不隨機**——具體條件 → 具體速度。

## III. 擁塞（Traffic Congestion）

### 同條道路多單位

戰役中常見場景：3AD 主力+補給車隊+傷員後送同走 A 級路。

**擁塞計算（每 hex 道路）：**

```
道路容量 = 道路級別容量值
  A 級：~3,000 vehicles/hour 通過
  B 級：~2,000 vehicles/hour
  C 級：~1,200 vehicles/hour
  農用路：~600 vehicles/hour

實際流量 = 該 hex 通過的車輛/部隊總數

擁塞係數 = 容量 / 流量
  > 1.0：道路順暢
  0.5-1.0：擁塞、速度 ×0.7
  0.2-0.5：嚴重擁塞、速度 ×0.4
  < 0.2：堵死、速度 ×0.1（基本停擺）
```

### 範例：3AD 全師通過 A 級路

- 3AD 全部車輛：~1,500 輛（戰車+半履帶+卡車+牽引砲+jeep）
- 通過時間：1,500 / 3,000 = 0.5 hour 通過 1 hex
- 整個 3AD 部隊通過 6 hex 道路 = 3 hour（不只是 6 hour）

**這就是裝甲師「移動慢」的真實原因**——不是車輛慢、是車隊長到道路擁塞。

### 擁塞惡化條件
- 雨後泥濘：道路容量 ×0.6
- 損壞道路：×0.5
- 戰鬥中部隊（車輛分散）：×0.4
- 補給車隊優先：擠掉戰鬥單位 40% 容量

## IV. 疲勞累積（Fatigue）

### 疲勞計算

每單位有「疲勞值」（0-100，初始 0）：

```
連續行軍每 hour 增加疲勞：
  正常天氣 +5
  雨天 +7
  泥濘 +10
  夜間 +3（額外）
  熱天（>30°C）+5（額外）
  
連續戰鬥每 hour 增加疲勞：
  輕戰鬥（探查）+8
  中戰鬥（預設陣地）+15
  重戰鬥 +25
  
休整每 hour 減少疲勞：
  完全休整 -10（睡覺）
  接戰待命 -3
  輕度活動 -1
```

### 疲勞效應

| 疲勞 | 效應 |
|------|------|
| 0-20 | 標準（無效應）|
| 20-40 | Org 上限 -5 |
| 40-60 | Org 上限 -10、移動 ×0.9 |
| 60-80 | Org 上限 -15、移動 ×0.8、戰鬥效力 ×0.9 |
| 80-100 | Org 上限 -25、移動 ×0.7、戰鬥效力 ×0.8 |

### 完整修整

- 撤離戰場 6 hour 休整：疲勞 -50%
- 撤離戰場 12 hour 完整休：疲勞清零
- 戰役期間每 3 天必須有 1 天休整、否則整師戰力長期下降

## V. 車輛/裝備故障

### 機械故障率（每 100 公里行軍）

| 車輛 | 故障率 |
|------|--------|
| M4 Sherman 75mm | ~3% |
| M4 Sherman 76mm | ~4% |
| M5 Stuart | ~2% |
| M7 Priest 自走砲 | ~5% |
| M8 Greyhound | ~2% |
| M3 半履帶 | ~3% |
| Pz IV Ausf. H | ~5% |
| Panther Ausf. G | **~12%**（**裝甲師最大隱患**）|
| Tiger I | ~15% |
| Stug III | ~4% |
| Opel Blitz 卡車 | ~4% |
| BMW R75 摩托 | ~6% |
| 各式 Jeep | ~2% |

**注：** Panther 與 Tiger 的高故障率是歷史事實——傳動系統設計缺陷、巨大重量、戰爭末期品控下降。

### 故障的處理

```
故障發生 → 進「待修」狀態
  輕度故障（傳動、引擎雜音）：2-4 hour 連級修
  中度故障（傳動齒輪、履帶斷）：6-12 hour 營級修、需 PARTS
  重度故障（引擎、傳動全壞）：24+ hour 軍級修、可能放棄
  
PARTS 池：
  美軍裝甲師：充足（修復 95% 故障）
  德軍裝甲師：嚴重短缺（戰役末段僅修 40% 故障）→ 剩餘戰車自損
```

### 範例：3AD 從 (5,5)F 北森林移動到 (14,3) 北橋

- 距離 ~6 hex × 2km = 12km（實際公路繞行可能 15-20km）
- 行軍故障：184 輛 M4 × 3% / 100km × 18km = ~10 輛故障
- 待修：4 輛輕度（2-4 hr 修）、4 輛中度（6-12 hr）、2 輛重度
- **預計戰場可用戰車數**：184 → 178（**Run 1 我估「7 輛 M4 散兵損失」太籠統，公式版更精確**）

### 範例：2Pz Panther 連從 (22,5) 機動到 (15,9) 反擊

- 距離 ~9 hex × 2km = 18km
- Panther 34 輛 × 12% / 100km × 18km = ~7 輛故障
- 加上戰鬥損失：抵戰場時 Panther 27-30 輛實際可戰
- **Run 1 Tick 5 Adler 2Pz「85→52」的折扣中、約 5 輛是機械故障、不只是戰鬥**

## VI. 河流與渡河

### 河流類型

| 河流 | 寬度 | 過河方式 |
|------|------|---------|
| 小溪（streams）| < 5m | 步兵涉水、車輛繞行 |
| 中型河（La Sève 寬窄不一） | 20-50m | **必須橋或浮橋** |
| 大河（Seine、Marne）| 100m+ | 工兵浮橋專案、大型行動 |

### 過橋容量（每 hex 橋）

| 橋型 | 容量 |
|------|------|
| 古橋（石橋、戰前）| ~2,000 vehicles/hour（A 級道路相容）|
| 木舟橋（工兵搭）| ~800 vehicles/hour |
| M2 浮橋（美軍標準）| ~1,200 vehicles/hour |
| B-Gerät（德軍標準）| ~1,000 vehicles/hour |
| 重型浮橋（軍級+）| ~1,800 vehicles/hour |

### 過橋戰術現實

- 1 個步兵團（~3,000 人 + 卡車 + 砲）過橋 ~ 1-2 hour
- 1 個裝甲師（1,500 輛）過橋 ~ 3-4 hour 一座橋（**極大瓶頸**）
- 過橋時被砲擊：傷亡上限可達 25%（密集隊形+橋頭瓶頸）
- 橋斷在中途過橋 = 喪失部隊（如 Run 1 北橋如果在 3AD 過河中被引爆）

### 浮橋搭建

| 浮橋類型 | 工兵需求 | 物資需求 | 完工時間 |
|---------|---------|---------|---------|
| M2 浮橋小型（30m 跨度）| 1 個工兵連 | 1 套器材（forces_v1 已列）| 6 hour |
| M2 浮橋標準（80m）| 1 個工兵營 | 1 套標準器材 | 12 hour |
| M2 浮橋重型（120m+）| 1 個工兵營+加強連 | 1 套+備用 | 18-24 hour |
| 90ID 浮橋專案在 (10,17) | 1 個工兵營+輕量套件 | 1 套輕量浮橋 | 12 hour（中斷則重來）|

## VII. 戰鬥行軍 vs 一般行軍

### 行軍狀態

| 狀態 | 描述 | 速度 | 接敵反應 |
|------|------|------|---------|
| **行軍縱隊**（March Column）| 部隊在道路上排成長條，無戰備 | ×1.0 | 反應慢 +30 分鐘部署 |
| **戰備行軍**（Tactical Movement）| 偵察排前出、戰備陣形 | ×0.7 | 反應快 5 分鐘可接戰 |
| **戰鬥行進**（Combat Advance）| 預期接敵、慢速進攻 | ×0.4 | 隨時可接戰 |
| **行軍中遇襲**（Marching Ambushed）| 縱隊被攻擊 | 部隊損失 +50%（被打散）|

### 戰術選擇後果

- 你下「3AD 急行軍至 (14,3)」= 行軍縱隊（快但脆弱）
- 你下「3AD 戰備推進」= 戰備（慢但安全）
- 1944 戰場常識：在敵後線附近用戰備、安全區用縱隊

## VIII. 補給卡車的特殊規則

詳細已於 `logistics_v1.md`，補充：

## IX. 範例：Run 1 移動實況重算

### 範例 1：3AD 從 (5,5)F 隱蔽到 (9,3)F（Run 1 Tick 3）

- 距離：4-5 hex 對角
- 路徑：(5,5)F → (6,4) plain → (7,3) plain → (8,3) plain → (9,3)F
- 步驟：
  1. (5,5)F → (6,4) plain：森林離開 0.5 hex（裝甲 ×0.5）= 1 hour
  2. (6,4) → (7,3) → (8,3)：每段平原 1 hex（裝甲 ×1.0 開闊）= 1 hour 各
  3. (8,3) → (9,3)F：進森林 0.5 hex = 1 hour
- 總時：~4 hour
- 但夜間 ×0.5 → 6-8 hour
- **Run 1 我說「Tick 3 進入 (9,3)F」是合理的**

### 範例 2：2Pz 從 (19,7) 反擊到 (15,9)（Run 1 Tick 6）

- 距離：4-5 hex
- 路徑：A 級路（中橋方向）
- 速度：道路 ×1.5 × 裝甲 1.0 = 1.5 hex/hour
- 機械故障：12% / 100km × 9km = ~1 輛故障
- 抵達時間：~3-4 hour
- **Run 1 我說「2Pz 從 (19,6) 移動到 (15,9) 區域大概 11:00 抵達」吻合**

### 範例 3：90ID 浮橋搭建（Run 1 Tick 2 起）

- 90ID 工兵營 + 輕量浮橋套件
- 標準完工 12 hour
- 但 Run 1 中：4-5 hour 後 352 工兵班突襲、損失 50% 進度
- 重來時間：6-8 hour 額外
- **Run 1 我說「需 1-2 tick 完工」太樂觀，公式版更悲觀**
